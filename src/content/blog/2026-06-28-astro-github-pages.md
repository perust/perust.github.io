개인 블로그를 만들 때 가장 중요한 기준은 오래 운영하기 쉬운가이다. 그래서 이번 사이트는 Astro와 GitHub Pages 조합으로 구성했다.

## 왜 Astro인가

Astro는 정적 사이트를 빠르게 만들기 좋다. 블로그 글은 Markdown으로 작성할 수 있고, 필요한 경우 컴포넌트를 붙여 포트폴리오와 블로그를 한 사이트 안에서 운영할 수 있다.

## 왜 GitHub Pages인가

GitHub Pages는 서버 비용 없이 정적 사이트를 배포할 수 있다. 개인 포트폴리오와 제작 기록을 공개하기에 충분하고, 커스텀 도메인도 연결할 수 있다.

## 수익화 준비 관점

AdSense나 제휴 링크를 고려한다면 다음 요소가 필요하다.

1. 독창적인 콘텐츠
2. 명확한 소개 페이지
3. 문의 페이지
4. 개인정보처리방침
5. 깨진 링크 없는 사이트 구조
6. 검색엔진이 읽기 쉬운 sitemap과 robots.txt

이번 개편에서는 이 기반을 먼저 넣었다.
